## 五、用户问题逐条回答

### 问题 1：ST-1 共用设备为什么在动画里 1号线、2号线都有车？

**答案 B：核心资源仍是共用单资源，只是动画按两线槽位显示。**

证据：
- ST-1 全量 1148 台车共享同一个 `resource_key = 引取::双线共用`
- 同资源加工重叠 = 0，前车 `svc_finish` 后、后车 `start` 才出现
- 动画代码（`_build_sim_track_layout`，line ~4118-4133）为**每个工位固定画 1号线/2号线两个槽位**，不像其他工位全根据 `_line_slots(line_scope)` 设置各槽 enable/disable

唯一的"同时出现"是 t=100s 时：
- Car#1 **已经离开 ST-1**（56s 完成），正在 ST-2 集约 1号线加工
- Car#2 正在 ST-1 2号线加工
- 视觉上看起来"ST-1 空闲位置上`还有1台`可能指向系统误判"

这是**视觉认知错误**——用户看到的是"两条线的槽位都有车"，但排程实际是"同一资源的时间交错"。

### 问题 2：共用设备是否发生同一时间两车**加工**？

**否。** 全量加工重叠 = 0 组。

### 问题 3：共用设备是否发生同一时间两车**占位**？

**否。** 全量占位重叠（`后车 start < 前车 depart`）= 0 组。所有对都满足 `后车 start ≥ 前车 depart`。

用整数秒检查时出现的"同时 2 台在位"（如 t=790s：Car#13 depart=790.0 & Car#14 start=790.0）属于时间步进的边界误判——Car#13 在 t=790s 时刻**已不在位**（开区间 `start ≤ t < depart` 不含右端点）。

### 问题 4：是否存在"前车等待进入 ST-2，后车已经在 ST-1 加工"？

**不存在。** 前 30 台全部满足 `ST-1 depart == ST-2 start`（严格交接），无隐形等待、无交叉加工。后车 ST-1 开始时刻始终晚于前车 ST-1 离开时刻。

### 问题 5：属于

**B. 动画显示 bug / 视觉歧义**——排程核心逻辑正确（单资源、无协调重叠、严格交替），但动画把共用设备画成两个独立槽位，给用户造成"共用设备在两线同时作业"的误解。

## 七、候选处理边界（仅建议，不改代码）

### 如果动画问题需要修复

1. **共用设备工位在动画上改为单槽位**：`_build_sim_track_layout` 对 `_station_device_mode == "shared"` 的工位只画一个跨越两线高度的槽位，不再画两个独立槽位。
2. **共用设备车辆按 `line_no` 仍可适当偏移**以表示"退出后进入哪条线"，但不画出两个独立加工位。
3. 对共用设备工位，单槽内至多显示一台车标签，交替使用透明度区分"前一秒离开"。当前"还有1台"标牌源自卡片内的槽位折叠控件而非动画布局。

### 如果动画暂不改动（当前优先级不要求动画修复）

1. 在"模型提示"或"使用说明"中增加注释："共用设备为单资源，动画上两条线的槽位不代表并行加工；该工位同一时刻只加工一台车。"
2. 在动画底部车道标签旁附加一句"两条线均只在加工时刻显示占用标签"。

### 如果后续发现现场共用设备的实际物理排布与模型假设冲突

需模型层面确认：共用设备是否确实为"一台设备，车辆从 1号线/2号线交替进入"，还是物理上有两个分线的独立设备。后者需将设备数调整为 2 并修正资源定义。
