## 四、16F报告验收记忆

报告字段必须满足：

1. `结果总览` 不显示达标车辆、达标率和最大相邻下线间隔前10项。
2. `车辆时间明细` 一车一行，按实际下线时间排序。
3. `截止统计时点已发生完工后等待(s)` 求和必须等于结果总览的累计实际等待。
4. `截止统计时点节拍外等待(s)` 求和必须等于结果总览的累计节拍外等待。
5. 比例模式必须区分 `分析窗口内 / 目标批次窗口外 / 仿真缓冲`。
6. `工程记录 SEGMENTS` 不自动换行。
7. 导出后必须能用 openpyxl 重新打开，并且四张表顺序固定。

## 五、16M动画验收记忆

共用设备显示要求：

1. `line_scope == "双线共用"` 的工位动画只显示一个跨两线单槽位。
2. 共用设备仍显示 `共用设备` 标签。
3. 共用设备车辆不按两条独立槽位绘制，只在单槽中显示。
4. 车辆标签用 `→1号线 / →2号线` 表示离开共用设备后的流向。
5. 普通双线双设备工位仍显示 1号线 / 2号线两个槽位。
6. 本轮只改动画层，不改排程、等待、报告和核心计算。

## 六、16L冻结序列导出/导入记忆

1. A276/B306/C566、最大连续5、交替混流的确定性序列SHA-256为 `dac6086f3b896255eeb77cb82679b0334e49ab972a5e9ed6aa0cd4e6d347f08f`。
2. 排序算法本身确定，不是随机排序；冻结导出/导入用于跨人员、跨电脑复现和留证。
3. 冻结文件schema为 `mline.freeze.sequence.v1`，默认后缀 `.mline-sequence.json`。
4. 导入必须硬校验程序版本、规则版本、A/B/C数量、总台数、最大连续台数、payload SHA-256和序列SHA-256。
5. 导入不会自动覆盖当前界面参数；界面参数必须与冻结文件一致。
6. 导入成功后旧分析结果立即失效，必须重新分析。
7. 16L只改冻结序列进出能力，不改排序算法、排程、等待、模型结果、报告或动画。
8. 2026-06-17 16K-B 轮已暂时隐藏 `导出冻结 / 导入冻结` 两个入口，只保留 `排列冻结`。冻结导出/导入实现保留，待 R9-B 引入 seed / 区间扰动后再恢复入口。

## 七、16K-B动画响应性记忆

1. 播放动画不再每帧 `clear()` 整个 `QGraphicsScene`。
2. 静态底图由 `_sim_static_signature` 控制，只有岗位、尺寸或槽位布局变化时重建。
3. 动态车辆层用 `_sim_dynamic_items` 记录并替换上一帧图元。
4. 加工、等待、转场事件通过 `_sim_event_cache_for_current_rows()` 预构建，tick 使用 `bisect` 查找当前时刻相关事件。
5. 输入变化、重新分析或分析结果失效时必须调用 `_invalidate_sim_scene_cache()`。
6. 验证入口：`ticket_export_app/scripts/verify_v29_round16kb_sim_incremental_render.py`。
7. 本轮只改 UI 动画响应性和冻结入口显示，不改 core、report、workflow、packaging。
