# M-Line 混流节拍仿真系统 使用说明 v2.9

## 1. 工具定位

M-Line 混流节拍仿真系统用于多车型混流排程仿真、工位能力节拍校核、整体节拍观察、阻塞风险提示和组合票导出辅助。

本工具不再输出 OK/NG 最终判定。程序提供模型结果和风险提示，使用者需要结合现场设备、人员、物流和管理经验进行最终判断。

## 2. 主界面说明

主界面保留多工程组合票和单工程组合票两个功能模块。

多工程组合票页面主要包含：

- 运行参数：设置工程名称、投车模式、车型比例或数量、分析时间、目标节拍和排列方式。
- 岗位矩阵：录入每个工位的设备数量、线别、设备名称和 A/B/C 车型工时。
- 车型数据摘要区：展示本次排程的基础摘要和模型结果。
- 仿真回放区：以暗色生产线视图回放车辆在工位和通道中的流转。
- 车辆明细 / 调试日志：按车辆维度查看每台车的完整工序链。

单工程组合票页面用于维护单工程作业行和导出辅助，不受仿真结果口径影响。

## 3. 运行参数说明

- 工程名称：本次排程或组合票对应的工程名称。
- 投车模式：支持按比例投车和按数量投车。
- A/B/C 比例或数量：按比例模式下填写车型比例；按数量模式下填写车型台数。
- 分析时间：按比例投车时用于计算仿真分析窗口。
- 目标节拍：用于工位能力节拍、达标车辆、整体节拍和风险提示的目标值。
- 排列方式：按数量投车时可选择顺排或交替混流。
- 最大连续台数：限制同一车型连续出现的最大台数。

## 4. 岗位矩阵说明

岗位矩阵字段包括：

- 序号：工位顺序。
- 工程名称：工位或工程名称。
- 设备数量：该工位可用设备数量。
- 所属线别：1号线、2号线、双线或双线共用。
- 岗位设备：设备或资源名称。
- A工时 / B工时 / C工时：对应车型在该工位的加工时间。

设备数量和所属线别会影响有效设备能力。双线双设备按并行能力 2 计算；单线单设备和双线共用设备按并行能力 1 计算。车型工时为 0 时，可理解为该车型跳过该工位。

## 5. 模型结果区口径说明

模型结果区显示客观指标，不显示最终 OK/NG 判定。

- 下线车辆：OUT <= 分析时间 的车辆数。
- 达标车辆：下线车辆中，经过的所有有效工位，其工位能力节拍均 <= 目标节拍的车辆数。
- 达标率：达标车辆 / 下线车辆。
- 工位能力节拍：当前车型该工位加工时间 / 该工位有效设备能力。
- 整体节拍：下线车辆整体 OUT 平均节拍，公式为 `(最后一台 OUT - 第一台 OUT) / (下线车辆数 - 1)`。
- 累计阻塞：仿真过程中累计等待 / 阻塞时间汇总。

当下线车辆少于 2 台时，整体节拍显示为 `-`。

## 6. 风险提示说明

风险提示包含三类信息：

- 工位能力超节拍：提示哪些工位按有效设备能力折算后仍超过目标节拍。
- 阻塞工程：提示实际排程中产生等待或阻塞的工程。
- 整体节拍：仅当整体节拍超过目标节拍时显示。

无风险时显示：

```text
暂无明显风险
```

风险提示不是最终判定，只用于提醒使用者关注可能需要现场复核的位置。

## 7. 实时节拍说明

实时节拍显示在仿真回放控制栏中。

实时节拍 = 当前已下线车辆最近最多 5 个 OUT 间隔平均。

实时节拍只用于播放过程观察，不参与达标车辆、达标率、风险提示和整体节拍计算。

## 8. 仿真回放说明

仿真回放区用于观察车辆在生产线中的流转。

- 播放：按当前速度推进仿真时间。
- 暂停：暂停仿真时间推进。
- 重置：回到仿真初始时间。
- 速度选择：可选择多个播放倍率。
- 车辆显示：车辆主体显示车号和车型色条。
- 加工车辆：显示在对应岗位槽位内。
- 等待车辆：使用红色强化显示，提示当前车辆存在等待。
- 移动车辆：按槽位、通道、目标槽位的路径进行视觉过渡。

仿真区不再根据单个工序工时大于目标节拍显示“超时”。

## 9. 车辆日志说明

车辆明细 / 调试日志以文本形式显示，每台车一行。主要字段包括：

- CAR：车辆编号。
- TYPE：车型。
- IN(s)：进入首工位时间。
- OUT(s)：完成末工位时间。
- WAIT(s)：累计等待。
- FLOW(s)：OUT - IN。
- RESULT：车辆工序观察结果。
- SEGMENTS：该车经过的完整工序链。

RESULT 是车辆工序观察字段，不代表模型最终判定。最终应以模型结果区和风险提示区为主。

## 10. 推荐使用流程

1. 填写运行参数。
2. 填写岗位矩阵。
3. 点击生成并导出组合票，或进入下一步分析与导出。
4. 查看车型数据摘要区和模型结果区。
5. 播放仿真，观察车辆流转、实时节拍和等待状态。
6. 查看风险提示和车辆日志。
7. 结合现场经验判断当前方案是否可接受。

## 11. 典型场景解释

例 1：双线双设备，工位能力满足。

目标节拍 58s，ST2 B 工时 116s，设备能力 2，则 `116 / 2 = 58`，不属于工位能力超节拍。

例 2：ST2 单设备瓶颈。

目标节拍 58s，ST2 B 工时 116s，设备能力 1，则 `116 / 1 = 116`，超过目标节拍。系统会提示工位能力超节拍，并可能出现阻塞工程和整体节拍风险。
