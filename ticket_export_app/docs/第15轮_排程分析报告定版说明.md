# 第15轮 排程分析报告定版说明

## 一、定版目标

将现有车辆日志、模型结果说明和总完成时间复核整合为一份简洁的 Excel 报告，避免程序继续增加重复弹窗。

定版报告名称：

`M-Line排程分析报告.xlsx`

报告只读取当前已经生成的排程结果，不重新运行排程，不改变任何计算口径。

## 二、报告结构

### 1. 结果总览

控制在一屏内，仅显示：

- 统计范围；
- 下线车辆；
- 批次总完成时刻；
- 最后完成车辆；
- 整体节拍；
- 累计实际等待；
- 累计节拍外等待；
- 三条时间真实性核对；
- 主要节拍外等待位置；
- 超节拍工程；
- 节奏净差值。

不显示达标车辆、达标率和最大相邻下线间隔前10项。

### 2. 车辆时间明细

一车一行，保留原车辆日志全部内容，并增加：

- 下线顺序；
- 前一台下线车辆；
- 相邻下线间隔；
- 与目标节拍差值；
- 延误增量；
- 提前补偿；
- 累计下线间隔；
- 单车等式差。

间隔与核对字段使用 Excel 公式，用户可点击单元格查看计算来源。

### 3. 计算口径说明

只解释人工复核所需概念：

- 批次总完成时刻；
- 单车贯通时间；
- 相邻下线间隔；
- 整体节拍；
- 车辆总等待；
- 累计实际等待；
- 累计节拍外等待；
- 延误增量与提前补偿；
- SEGMENTS。

## 三、三条时间核对

```text
单车：投车时间 + 单车贯通时间 = 下线时间
批次：第一台下线时间 + 全部相邻下线间隔 = 最晚下线时间
总完成时间：目标批次全部车辆下线时间的最大值
```

累计等待已经体现在车辆工程离开时间和下线时间中，不得再次加到总完成时间。

## 四、程序界面入口（第16F轮定版）

模型结果标题行右侧设置：

```text
模型结果                                  [导出分析报告]
```

### 导出分析报告

- 作为模型结果区唯一的报告入口；
- 尚未完成分析时禁用；
- 分析完成后启用；
- 导出当前分析对应的完整 Excel 报告；
- 按钮提示直接说明报告含有结果总览、车辆时间明细和计算口径说明；
- 报告直接读取当前分析快照，不重新运行排程；
- 不放在车辆日志窗口中；
- 车辆日志窗口继续保留界面筛选和“导出当前筛选”CSV。

第16轮审核后取消独立“结果说明”和“报告说明”弹窗。主界面只保留当前决策所需摘要，完整计算口径、人工复算公式和逐车时间证据统一进入分析报告，避免同一内容在主界面、弹窗和报告中重复维护。

## 五、实现边界

后续实现只整理和导出现有排程结果，不修改：

- 排程顺序；
- 资源服务顺序；
- 投车时间；
- 等待时间；
- 下线时间；
- 整体节拍；
- 风险计算。

第15轮实施前后，真实场景排程行和分析结果必须完全一致。
