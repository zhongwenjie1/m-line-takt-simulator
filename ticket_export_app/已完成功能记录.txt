四、当前逻辑边界记录

1. 当前“目标节拍”只用于 OK/NG 判定

2. 当前等待时间是“资源等待”

当前等待时间表示：
车辆因为前方资源未释放、不能进入下一步骤而产生的等待。

当前等待时间不是：
因为目标节拍超出而产生的节拍等待。


3. 当前不是“固定投车节拍模型”

当前程序不是按以下逻辑排程：
第 n 台车理论进入时间 = (n - 1) × 投车节拍。

后续如果做“投车节拍模型”，才会出现：
理论投车时间 + 资源释放时间 → 共同决定实际开始时间。

该点已记录为后续迭代。


4. 当前按比例投车逻辑已完成第一阶段重构

当前规则：
投车模式分为两类：
- 按数量投车

按数量投车：

当前 v2-6 核心分析口径（已收口为“下线口径”）：
- 计划下线台数 = 空线起步口径下，规定时间内按目标节拍理论可下线的整车台数
- 理论首车下线时间 = 工程数 × 目标节拍
- 若分析时间 < 理论首车下线时间：计划下线台数 = 0
- 否则：计划下线台数 = floor((分析时间秒数 - 理论首车下线时间) / 目标节拍) + 1
- 真实实际下线台数 = 分析时间内实际完成全部工程的车辆数量
- 显示实际下线台数 = min(真实实际下线台数, 计划下线台数)，正常达成时不额外显示超额下线数量，避免产生疑问
- 达成率 = min(真实实际下线台数 / 计划下线台数, 100%)
- 实际下线节拍 = 参与统计车辆的下线时间间隔平均值
  - 若真实实际下线台数 >= 计划下线台数：统计第 1 台到第计划下线台数台
  - 若真实实际下线台数 < 计划下线台数：统计第 1 台到最后一台实际下线车辆
- 累计阻塞、超节拍工程只作为风险提示显示，不直接参与最终判定

当前模型判定显示格式：
最终判定：OK / NG
下线达成：计划xxx台｜实际xxx台｜达成率：xx.x%
实际下线节拍：xxx.xs/台｜目标节拍xxxs/台
累计阻塞：xxs｜超节拍工程：xxx

当前最终判定规则：
- 真实实际下线台数 >= 计划下线台数 → OK
- 真实实际下线台数 < 计划下线台数 → NG
- 累计阻塞、超节拍工程、实际下线节拍只作为风险/参考信息，不直接覆盖最终判定

当前需要避免的旧口径：
- 不再使用“理论投车台数”作为达成率分母
- 不再使用“工程段折算台数”作为 v2-6 主达成率分母
- 不再使用“已进入工程段数”判断实际产能达成
- 不再用“总分析时间 / 实际下线台数”计算达成时的实际下线节拍，避免把窗口尾差计入节拍
- 不再让累计阻塞或超节拍工程直接覆盖最终判定

八、排程模型 v2 已达成共识

1. 固定投车节拍

固定投车节拍直接引用现有“目标节拍 / 数据节拍时间”，不再新增用户输入参数。

理论投车时间：
第 n 台车理论投车时间 = (n - 1) × 目标节拍

用于分析持续投车时：
- 实际下线时间是否滞后
- 给定时间窗口内少下线多少台
- 累计等待时间可折算为多少台车


2. 设备数量 + 所属线别替代原“运行方式”

原字段“运行方式”与“所属线别”含义重叠，后续 v2 简化为：
- 设备数量
- 所属线别

最终岗位矩阵 v2 字段建议为：
序号 / 工程名称 / 设备数量 / 所属线别 / 岗位设备 / A工时 / B工时 / C工时

设备数量：
- 1：该岗位同一时间只能处理 1 台车
- 2：该岗位可支持双线并行处理

所属线别：
- 1号线
- 2号线
- 双线
- 双线共用

组合规则：
- 设备数量 = 1，所属线别 = 1号线：该岗位只在 1号线，适用车型必须进入 1号线
- 设备数量 = 1，所属线别 = 2号线：该岗位只在 2号线，适用车型必须进入 2号线
- 设备数量 = 1，所属线别 = 双线共用：两条线都可到达，但设备共用一套，同一时间只能处理 1 台车
- 设备数量 = 2，所属线别 = 双线：1号线和2号线各有一套设备，可并行处理

第一版建议限制：
- 设备数量 = 1 时，所属线别可选：1号线 / 2号线 / 双线共用
- 设备数量 = 2 时，所属线别固定为：双线


3. A/B/C 工时同时表达“是否需要该岗位”

取消“适用车型”字段。

后续规则：
- 工时 > 0：该车型需要经过该岗位，工时为填写值
- 工时 = 0：该车型跳过该岗位
- 空白：只允许在该车型运行参数数量 / 比例为 0 时出现

重要规则：
参与投车的车型，工时必须填写。
如果某车型数量 / 比例 > 0，则该车型在岗位矩阵每一行的工时不能为空。

空白不再表示继承 A 工时。
空白只表示该车型当前未参与投车，不参与计算。

示例：
A/B 参与投车，C 不参与投车：
- A工时、B工时必须填写
- C工时可以为空

示例：
A/B/C 都参与投车：
- A工时、B工时、C工时都必须填写
- 若某车型不需要某岗位，应填写 0，而不是空白


4. 特定岗位只约束需要该功能的车辆

若某条线因前车必须进入特定岗位而发生堵塞，后车若不需要该特定岗位，不应被该堵塞强制阻断。

后车处理规则：
- 如果不需要该特定岗位，可以自动选择其他预计最早完成的线别继续
- 如果仍在有特定岗位的线别上，也可以在前车完成该特定岗位后，直接跳过该岗位进入下一工程

核心原则：
特定岗位只约束需要该功能 / 设备的车辆。
不需要该功能 / 设备的车辆可以跳过，不必等待完成该特定岗位。


5. 线别选择原则

车辆进入线别时，按以下优先级判断：

第一步：判断该车型在该岗位的工时是否为 0。
- 如果为 0：跳过该岗位
- 如果 > 0：该车型需要该岗位

第二步：根据设备数量与所属线别判断资源规则。
- 1号线：必须进入 1号线
- 2号线：必须进入 2号线
- 双线：自动选择预计最早释放 / 最早完成的线别
- 双线共用：可从双线接入，但设备资源共用一套

第三步：对于无指定线别约束的车辆，选择预计最早释放 / 最早完成的线别。


6. 溢出分析范围

当前 v2-6 已调整并收口为“规定时间窗口下线达成”口径：

B：给定时间窗口内的计划下线与实际下线差异
- 计划下线台数 = 空线起步口径下，规定时间内按目标节拍理论可下线的整车台数
- 真实实际下线台数 = 规定时间内实际完成全部工程的车辆数量
- 显示实际下线台数 = min(真实实际下线台数, 计划下线台数)
- 达成率 = min(真实实际下线台数 ÷ 计划下线台数, 100%)
- 实际下线节拍 = 参与统计车辆的下线时间间隔平均值

C：累计等待时间 / 累计阻塞
- 累计等待时间继续显示为风险提示
- 当前最终判定不再直接使用累计等待时间折算台数

说明：
理论投车 / 投车生成只用于生成足够车序，不作为最终达成率分母。
超节拍工程只作为风险提示显示，不直接覆盖最终判定。


九、排程模型 v2 开发计划

v2-0：岗位矩阵 v2 数据结构改造

目标：
将岗位矩阵字段调整为：
序号 / 工程名称 / 设备数量 / 所属线别 / 岗位设备 / A工时 / B工时 / C工时

主要事项：
- 删除原“运行方式”字段
- 不新增“适用车型”字段
- 不新增“是否强制线别”字段
- 增加“设备数量”字段
- 保留“所属线别”字段，但含义重新定义
- 明确 A/B/C 工时 = 是否经过该岗位 + 作业时间
- 增加工时空白校验规则

验证目标：
用户能通过设备数量、所属线别、A/B/C 工时，完整表达：
- 普通双线双设备
- 单线独有设备
- 双线共用单设备
- 某车型跳过某岗位
- 某车型必须进入某条线完成特定岗位


v2-1：固定投车节拍参与排程

目标：
使用现有目标节拍作为固定投车节拍。

核心公式：
理论投车时间 = (车序号 - 1) × 目标节拍
实际进入首岗位时间 = max(理论投车时间, 首岗位资源释放时间)

验证场景：
- 工时 100s，目标节拍 118s：不应产生节拍溢出
- 工时 120s，目标节拍 118s：观察是否产生累积影响
- 工时 200s，目标节拍 118s：应明显产生溢出
- A/B 混流 100s / 120s：验证短工时是否能抵消部分超节拍影响


v2-2：车型跳过岗位逻辑

目标：
通过 A/B/C 工时是否为 0，判断车型是否跳过岗位。

规则：
- 工时 > 0：进入岗位
- 工时 = 0：跳过岗位
- 参与投车车型工时为空：报错
- 未参与投车车型工时为空：允许

验证场景：
A/B 参与投车，C 不参与投车：
- C工时允许为空
- A/B 工时必须填写

A/B/C 都参与投车：
- A/B/C 工时都必须填写
- 不需要某岗位的车型必须填写 0


v2-3：设备数量与线别资源模型

目标：
根据设备数量和所属线别计算资源占用。

资源规则：
- 设备数量 1 + 1号线：单线独有资源
- 设备数量 1 + 2号线：单线独有资源
- 设备数量 1 + 双线共用：双线共享单资源
- 设备数量 2 + 双线：双线独立双资源

验证场景：
- B 车型必须进入 1号线完成空悬+快充，A/C 跳过
- 双线共用单设备时，同一时间只能处理 1 台车
- 双线双设备时，1号线与2号线可并行处理


v2-4：自动择线模型

目标：
对于没有被单线独有设备约束的车辆，按预计最早释放 / 最早完成选择线别。

规则：
- 必须进入 1号线 / 2号线的岗位，按所属线别强制进入
- 双线岗位，选择预计最早完成线别
- 双线共用岗位，线别可选，但设备资源共用一套

验证场景：
1号线因前车进入特定岗位产生堵塞时：
- 需要该特定岗位的车继续等待该线
- 不需要该特定岗位的车可跳过该岗位
- 不需要该特定岗位的车可选择其他预计最早完成线别继续


v2-5：溢出分析

当前已采用的分析指标：
- 分析时间
- 目标节拍
- 计划下线台数
- 真实实际下线台数
- 显示实际下线台数
- 达成率
- 实际下线节拍
- 累计阻塞 / 累计等待
- 超节拍工程

当前计算规则：
计划下线台数 = 空线起步口径下，规定时间内按目标节拍理论可下线的整车台数
真实实际下线台数 = 分析时间内实际完成全部工程的车辆数量
显示实际下线台数 = min(真实实际下线台数, 计划下线台数)
达成率 = min(真实实际下线台数 ÷ 计划下线台数, 100%)
实际下线节拍 = 参与统计车辆的下线时间间隔平均值

待后续视需要补充：
真实超额下线数量 = 真实实际下线台数 - 计划下线台数（当前界面暂不显示）
等待折算台数 = 累计等待时间 ÷ 目标节拍（当前不参与最终判定）
